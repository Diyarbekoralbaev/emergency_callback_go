# Команды CLI

Единый бинарный файл выбирает действие по первому аргументу.

```
emergency-callback <command> [args]
```

| Команда | Назначение |
|---------|---------|
| `web` | Запуск HTTP-сервера (UI + API). |
| `worker` | Запуск фонового обработчика заданий (звонки, SMS, очистка). |
| `createuser` | Создание пользователя. |
| `seed` | Вставка демонстрационных регионов/бригад. |
| `migrate` | Запуск миграций приложения (goose). |
| `help` | Справка по использованию. |

Все команды читают `.env` из рабочего каталога.

---

## `web`

```bash
./emergency-callback web
```

Запускает HTTP-сервер на `HTTP_ADDR`. Подключается к базе данных и к River
(в режиме только очереди — он ставит задания в очередь, но не обрабатывает их). В продакшене
запускается под systemd ([Запуск сервисов](../operations/running-services.md)).

## `worker`

```bash
./emergency-callback worker
```

Запускает worker River: регистрирует три типа заданий и периодическую
очистку, затем обрабатывает задания до остановки. **Обязателен** для того, чтобы любой звонок или SMS
действительно состоялись. Параллелизм = `RIVER_MAX_WORKERS`.

Задания:

- `ProcessCallback` — выполняет один звонок через AMI.
- `SendRatingSMS` — отправляет резервное SMS.
- `CleanupStaleCalls` — периодический (каждые 15 минут) финализатор для зависших звонков.

## `createuser`

```bash
./emergency-callback createuser <username> <password> [admin|operator]
```

Создаёт пользователя с паролем, хэшированным через bcrypt. Если роль не указана, по умолчанию
используется `operator`. `admin` также устанавливает `is_staff`/`is_superuser`.

```bash
./emergency-callback createuser admin 'StrongPass!' admin
./emergency-callback createuser dispatcher1 'StrongPass!' operator
```

## `seed`

```bash
./emergency-callback seed
```

Вставляет два демонстрационных региона и четыре демонстрационные бригады (принадлежащие существующему администратору). Запускается
один раз после создания администратора, для быстрого демонстрационного/тестового набора данных. В продакшене можно безопасно пропустить.

## `migrate`

```bash
./emergency-callback migrate <up|down|status|version|reset>
```

| Подкоманда | Действие |
|-----------|--------|
| `up` | Применить все ожидающие миграции. |
| `down` | Откатить последнюю миграцию. |
| `status` | Показать применённые/ожидающие миграции. |
| `version` | Показать текущую версию схемы. |
| `reset` | Откатить все миграции (**деструктивно**). |

При необходимости переопределите каталог миграций через `MIGRATIONS_DIR`.

!!! note "Миграции River отдельны"
    `migrate` управляет только схемой **приложения**. Таблицы очереди заданий
    управляются через River CLI: `river migrate-up --database-url "$DATABASE_URL"`.
