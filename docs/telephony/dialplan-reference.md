# Справочник по dialplan

Приложение управляет вызовами через Asterisk: оно **инициирует** (originate) Local-канал, а затем выполняет действия **Redirect** через AMI. Задействованы четыре контекста. На FreePBX контекст `from-internal` уже существует (не переопределяйте его); вы добавляете остальные три.

## Как соединяются плечи вызова

Когда worker инициирует `Local/<number>@from-internal/n` с параметром `Context=ambulance-callback`, Asterisk создаёт Local-канал из двух половин:

- **Плечо `;1`** выполняет `from-internal` → ваш исходящий маршрут → trunk → телефон вызываемого.
- **Плечо `;2`** — это управляющее плечо → выполняет `ambulance-callback,s` **после того, как вызываемый ответит**.

Эти два плеча соединены (bridged), поэтому вызываемый слышит всё, что воспроизводит плечо `;2`. Суффикс `/n` не даёт Local-каналу «оптимизироваться» (optimize away), что необходимо, чтобы приложение могло выполнить `Redirect` плеча `;2` в `play-audio` после ответа.

Переменные, задаваемые в момент originate (наследуются обоими плечами):

| Переменная | Значение |
|----------|---------|
| `CALL_ID` | Внутренний UUID этого вызова в приложении. Передаётся обратно в каждом `UserEvent`, чтобы приложение могло сопоставить события с вызовом. |
| `PHONE_NUMBER` | Исходный номер телефона. |
| `BRIGADE_ID` | ID бригады. |
| `CALLBACK_REQUEST_ID` | ID строки в БД для этого обратного вызова. |

---

## `from-internal` (принадлежит FreePBX)

На FreePBX вы **не** определяете этот контекст — это стандартный контекст FreePBX, который применяет ваши исходящие маршруты (Outbound Routes) и отправляет вызов в trunk. Приложение набирает в нём **локальный номер с отсечённым префиксом**.

На **автономном** Asterisk (без FreePBX) вы должны предоставить его сами; см. [Автономный Asterisk](standalone-asterisk.md).

## `ambulance-callback`

Управляющее плечо. Выполняется только после того, как вызываемый ответит.

```ini
[ambulance-callback]
exten => s,1,NoOp(ANSWERED CALL_ID=${CALL_ID} PHONE=${PHONE_NUMBER})
 same => n,Answer()
 same => n,UserEvent(CallAnswered,CallID: ${CALL_ID},Phone: ${PHONE_NUMBER})
 same => n,Wait(300)
 same => n,Hangup()
```

| Строка | Назначение |
|------|---------|
| `Answer()` | Отвечает на управляющее плечо. |
| `UserEvent(CallAnswered,…)` | Генерирует AMI-событие, которое слушает приложение. Приложение захватывает **этот канал** как тот, который нужно перенаправлять, и запускает запрос оценки. |
| `Wait(300)` | Удерживает плечо активным, пока приложение управляет им через Redirect. Когда завершить вызов, решает приложение, а не эта строка. |

## `play-audio`

Воспроизводит именованную подсказку, затем ждёт DTMF. Приложение перенаправляет сюда вызов, при этом **в качестве extension указывается имя аудио** (например, `ambulance-rating-request`).

```ini
[play-audio]
exten => _.,1,NoOp(PLAY ${EXTEN} CALL_ID=${CALL_ID})
 same => n,Playback(${EXTEN})
 same => n,UserEvent(AudioPlayed,CallID: ${CALL_ID},Audio: ${EXTEN})
 same => n,WaitExten(60)
 same => n,Wait(60)
 same => n,Hangup()
```

| Строка | Назначение |
|------|---------|
| Шаблон `_.` | Совпадает с любым extension — то есть с любым именем аудио. |
| `Playback(${EXTEN})` | Воспроизводит WAV-файл, имя которого равно extension. |
| `WaitExten(60)` / `Wait(60)` | Удерживает плечо активным, чтобы вызываемый мог нажимать клавиши; приложение перехватывает события `DTMFEnd`. Обычно приложение снова перенаправляет вызов (на благодарность) или завершает его до того, как истекут эти таймауты. |

Имена аудио, которые использует приложение: `ambulance-rating-request`, `ambulance-rating-thankyou`, `ambulance-rating-invalid`. См. [Аудиоподсказки](audio-prompts.md).

## `transfer-to-337`

Куда уходит вызов, когда вызываемый выбирает перевод на оператора (нажимает `0` или `9`).

```ini
[transfer-to-337]
exten => s,1,NoOp(TRANSFER CALL_ID=${CALL_ID})
 same => n,Dial(Local/337@from-internal,30)
 same => n,Hangup()
```

Измените цель `Dial(...)` на ваше назначение оператора (extension, ring group или queue). Имя контекста — историческое (`337`); **имя не обязано совпадать с назначением** — совпадать с именем этого контекста должна только цель redirect в приложении.

!!! note "Имя контекста — это контракт"
    Приложение перенаправляет вызов на буквальные имена контекстов `play-audio` и `transfer-to-337`, а инициирует вызов в `ambulance-callback`. Если вы переименуете контекст в dialplan, приложение перенаправит вызов в несуществующий контекст, и вызов сломается. Сохраняйте эти три имени в точности как показано (их переименование требует изменения приложения).

## Что отправляет приложение (действия AMI)

| Момент | Действие AMI |
|--------|-----------|
| Начало вызова | `Originate` Local-канала в `ambulance-callback` |
| Воспроизведение любой подсказки | `Redirect` отвеченного канала → `play-audio`, ext `<audio-name>` |
| Перевод | `Redirect` → `transfer-to-337`, ext `s` |
| Завершение вызова | `Hangup` канала |

## Что слушает приложение (события AMI)

| Событие | Для чего используется |
|-------|----------|
| `Newchannel` | Первичное обнаружение канала (резервный механизм). |
| `UserEvent: CallAnswered` | Отмечает ответ; захватывает управляющий канал; запускает запрос оценки. |
| `DTMFEnd` | Нажатия клавиш вызываемым (оценка 1–5, перевод 0/9). Дублирующиеся цифры, приходящие по нескольким плечам моста, дедуплицируются. |
| `Hangup` | Завершает вызов. |
