# Конфигурация

Вся конфигурация хранится в одном файле `.env` в рабочем каталоге
бинарника. Он загружается при запуске. Краткая таблица приведена в разделе
[Переменные окружения](../reference/environment-variables.md); на этой странице
поясняется каждое значение и **что изменится, если вы его измените**.

!!! warning "Синтаксис `.env`"
    Обычные пары `KEY=value`, по одной на строку. Окружающие кавычки не нужны, а
    специальные символы без кавычек (`<`, `>`, пробелы) **ломают разбор**. Сломанная
    строка приводит к тому, что последующие переменные выглядят «отсутствующими», и
    приложение аварийно завершается при запуске с ошибкой
    `required env var ... not set`.

## База данных

```bash
DATABASE_URL=postgres://ecb:PASSWORD@127.0.0.1:5432/emergency_callback?sslmode=disable
DB_POOL_MAX_CONNS=100
DB_POOL_MIN_CONNS=10
```

- **`DATABASE_URL`** — полный DSN PostgreSQL. Используется веб-сервером, worker,
  миграциями и River. Если он неверный, при запуске вы получите
  `password authentication failed` или ошибки подключения.
- **`DB_POOL_MAX_CONNS` / `DB_POOL_MIN_CONNS`** — границы пула подключений pgx.
  Увеличьте максимум, если запускаете много параллельных worker; держите его ниже
  `max_connections` в PostgreSQL.

## HTTP-сервер

```bash
HTTP_ADDR=:8000
SITE_DOMAIN=https://your-public-domain.example
SESSION_SECRET=<32+ random bytes>
CSRF_KEY=<exactly 32 random bytes>
```

- **`HTTP_ADDR`** — адрес прослушивания. За обратным прокси привязывайтесь к
  локальному порту (например, `127.0.0.1:8000`). Если порт занят, сервер выводит в
  лог `address already in use` и завершает работу.
- **`SITE_DOMAIN`** — публичный базовый URL. **Используется для построения ссылок
  голосования в SMS** (`<SITE_DOMAIN>/vote/<uuid>`). Если он неверный, получатели
  SMS получат недоступную ссылку. Без завершающего слэша.
- **`SESSION_SECRET`** — ключ шифрования для сессионных cookie. **Его изменение
  разлогинит всех** (существующие сессии становятся недействительными).
- **`CSRF_KEY`** — должен быть **ровно 32 байта**. Его изменение делает
  недействительными отправляемые в данный момент формы (пользователям просто нужно
  перезагрузить страницу). Сгенерировать можно командой
  `openssl rand -base64 24`.

## Asterisk AMI

```bash
AMI_HOST=127.0.0.1
AMI_PORT=5038
AMI_USERNAME=ecb
AMI_SECRET=<secret>
AMI_CALLER_ID=781138081
AMI_OPERATOR_QUEUE=777
AMI_CALL_TIMEOUT=60
AMI_RATING_RETRY_LIMIT=3
AMI_RATING_TIMEOUT=10
```

- **`AMI_HOST` / `AMI_PORT`** — где слушает AMI Asterisk. Обычно
  `127.0.0.1:5038`, когда worker работает на хосте PBX.
- **`AMI_USERNAME` / `AMI_SECRET`** — должны совпадать с пользователем AMI,
  которого вы создаёте в Asterisk
  ([Интеграция с FreePBX](../telephony/freepbx-integration.md)). Неверные
  учётные данные → worker не может инициировать звонки.
- **`AMI_CALLER_ID`** — номер caller ID, отображаемый при исходящем звонке.
  Держите его как простой номер/строку (см. предупреждение о разборе выше).
- **`AMI_OPERATOR_QUEUE`** — подсказка о назначении оператора (используется вашим
  dialplan перевода; приложение передаёт звонки в контекст `transfer-to-337`).
- **`AMI_CALL_TIMEOUT`** — количество секунд, которые worker ждёт завершения
  звонка, прежде чем отказаться от него. Строка в БД затем становится
  `failed`/`no_rating`.
- **`AMI_RATING_RETRY_LIMIT`** — сколько неверных нажатий клавиш допускается,
  прежде чем звонок прекратит запрашивать оценку.
- **`AMI_RATING_TIMEOUT`** — количество секунд ожидания ввода оценки
  (зарезервировано для настройки; время ожидания также регулируется dialplan).

## Eskiz SMS

```bash
ESKIZ_EMAIL=<email>
ESKIZ_PASSWORD=<password>
ESKIZ_BASE_URL=https://notify.eskiz.uz/api
ESKIZ_DRY_RUN=false
```

- **`ESKIZ_EMAIL` / `ESKIZ_PASSWORD`** — учётные данные аккаунта Eskiz. Клиент
  выполняет вход, кэширует bearer-токен и автоматически обновляет его при `401`.
- **`ESKIZ_BASE_URL`** — базовый адрес API; меняется редко.
- **`ESKIZ_DRY_RUN`** — `true` записывает в лог предполагаемое SMS вместо его
  отправки. Отлично подходит для тестирования резервного сценария без оценки, не
  тратя кредиты и не отправляя сообщения реальным людям.

## Worker

```bash
RIVER_MAX_WORKERS=5
```

- **`RIVER_MAX_WORKERS`** — максимальное число параллельных задач, обрабатываемых
  worker. Каждый активный звонок удерживает одно подключение AMI и один канал
  Asterisk, поэтому подбирайте это значение исходя из ёмкости каналов вашего
  trunk. См. [Резервное копирование и масштабирование](../operations/backups-scaling.md).

## Применение изменений

Конфигурация читается **при запуске**. После редактирования `.env` перезапустите
затронутый процесс:

```bash
sudo systemctl restart emergency-callback-web emergency-callback-worker
```

Горячей перезагрузки нет.
