# JSON API

Единственная публичная конечная точка позволяет внешним системам создавать
обратный вызов программно (например, диспетчерская система инициирует обратный
вызов сразу после выезда скорой помощи).

## Создание обратного вызова

```
POST /api/create/
Content-Type: application/json
```

Тело запроса:

```json
{ "phone_number": "+998901234567" }
```

| Поле | Обязательно | Примечания |
|-------|----------|-------|
| `phone_number` | да | Международный формат (`+998…`). |

### Поведение

- **Случайная активная бригада** назначается автоматически.
- Обратный вызов создаётся, и задача `ProcessCallback` ставится в очередь в той же
  транзакции БД; worker немедленно начинает набор.
- Конечная точка **освобождена от CSRF** и **не требует аутентификации** —
  защитите её на сетевом уровне (firewall / allowlist обратного прокси), если она
  не должна быть публичной.

### Успешный ответ

```json
{
  "success": true,
  "callback_id": 123,
  "phone_number": "+998901234567",
  "team": "Бригада 2",
  "region": "Каракалпакстан",
  "status": "pending",
  "message": "..."
}
```

### Ответы с ошибками

| HTTP | Тело | Причина |
|------|------|-------|
| 400 | `{"error": "..."}` | Отсутствует/некорректный `phone_number` или некорректный JSON. |
| 500 | `{"error": "..."}` | Нет активных бригад либо ошибка сервера/базы данных. |

## Пример

```bash
curl -X POST http://127.0.0.1:8000/api/create/ \
  -H 'Content-Type: application/json' \
  -d '{"phone_number":"+998901234567"}'
```

## Примечания

- Чтобы выбрать конкретную бригаду (а не случайную), создайте обратный вызов через
  форму в админ-интерфейсе — API по своей конструкции выбирает случайную активную
  бригаду.
- Применяется тот же жизненный цикл вызова, что и для обратных вызовов, созданных
  через UI; см. [Поток вызова](../telephony/call-flow.md).
