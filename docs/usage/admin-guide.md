# Руководство администратора

Администраторы имеют полный доступ: панель управления, аналитика оценок, управление
бригадами/регионами, экспорт в Excel и всё, что доступно операторам. Интерфейс
администратора на **русском** языке.

Вход выполняется по адресу `/users/login/`. После входа вы попадаете на панель управления.

## Навигация (боковая панель)

| Пункт | Путь | Кто видит |
|------|------|-------------|
| Панель управления (Dashboard) | `/` | admin |
| Новый вызов (New callback) | `/callbacks/create/` | admin, operator |
| Все вызовы (All callbacks) | `/callbacks/` | admin, operator |
| Регионы (Regions) | `/teams/regions/` | admin |
| Бригады (Teams) | `/teams/` | admin |
| Оценки (Ratings) | `/ratings/` | admin |

## Панель управления (`/`)

Отфильтрованный обзор активности:

- **Фильтры:** регион, бригада, диапазон дат. По умолчанию — сегодня.
- **Статистика вызовов:** всего, завершённые (включая переведённые), неудавшиеся, без оценки.
- **Статистика оценок:** всего оценок, среднее, доля успешных/неудачных, распределение
  1–5.
- **Эффективность бригад/регионов** и список **последних вызовов**.
- **Экспорт в Excel** (`/export-excel/`) — скачивает сводную таблицу по бригадам
  с учётом текущих фильтров.

## Создание вызова (`/callbacks/create/`)

1. Выберите **регион**, затем **бригаду**.
2. Введите **номер телефона** в международном формате, например `+998901234567`.
3. Отправьте форму. Система создаёт запись, и worker немедленно начинает набор.

!!! tip "Формат номера телефона"
    Введите `+998…`. Система нормализует его (отбрасывает код страны `998`)
    перед передачей в dialplan. См.
    [Поток вызова](../telephony/call-flow.md).

Вызов появляется в разделе **Все вызовы** с актуальным статусом.

## Список вызовов (`/callbacks/`)

- Фильтрация по статусу, бригаде, региону, поиску (телефон/бригада) и диапазону дат.
- В каждой строке отображаются телефон, бригада/регион, бейдж статуса, звёзды оценки (если есть),
  время создания и длительность.
- Щёлкните по строке, чтобы открыть страницу **деталей** (`/callbacks/<id>/`) с полным таймингом,
  оценкой (если собрана), сообщением об ошибке (если вызов неудачен) и URL для SMS-голосования.

Значения статусов описаны в разделе [Статус вызова](../reference/call-status.md).

## Оценки (`/ratings/`)

Аналитика по собранным оценкам: итоги, среднее, доля «хороших» оценок (4–5★)
и фильтруемый список (регион, бригада, значение оценки, дата). Используйте его для
анализа качества обслуживания по бригадам/регионам за период.

## Регионы и бригады

Иерархия диспетчеризации — **Регион → Бригада**. Каждый вызов привязан к
бригаде, а бригада принадлежит региону.

=== "Регионы (`/teams/regions/`)"

    - **Список** показывает код, название, число активных/всех бригад, статус.
    - **Создание / Редактирование** задают название, код, описание, флаг активности.
    - **Детали** перечисляют бригады в регионе.
    - **Удаление** удаляет регион (и каскадно — его бригады; подтверждайте
      внимательно).

=== "Бригады (`/teams/`)"

    - **Список** с поиском и фильтром по региону; переключатель показа неактивных бригад.
    - **Создание / Редактирование** задают название, регион, описание, флаг активности.
    - Неактивные бригады исключаются из новых вызовов и панелей управления.

## Пользователи и роли

- **admin** — всё перечисленное выше.
- **operator** — может только создавать вызовы и просматривать список вызовов.

Пользователи создаются из командной строки (экрана управления пользователями
в приложении нет):

```bash
./emergency-callback createuser <username> <password> [admin|operator]
```

См. [Команды CLI](../reference/cli-commands.md).
